# Drive CLI (`drv`)

Use this skill when you need to authenticate, configure kubeconfig, manage secrets, or log into Drive registries.

## Quick start
1. `drv auth` to authenticate via GitHub device flow.
2. `drv doctor` to validate the environment and fix issues.
3. `drv kubeconfig setup` to configure cluster access.

## Read when
- **You need auth or doctor**: `resources/auth-and-doctor.md`
- **You need kubeconfig setup**: `resources/kubeconfig.md`
- **You need registry login or mise setup**: `resources/registry-and-mise.md`
- **You need secret commands**: `resources/secrets.md`
- **You need env vars and runtime paths**: `resources/env-and-paths.md`
